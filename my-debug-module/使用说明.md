# Spring Session 源码阅读调试模块使用说明

## 概述

这个 `my-debug-module` 模块是专门为阅读Spring Session 1.0.0源码而创建的调试和测试环境。它提供了最简单的测试程序，帮助你理解Spring Session的核心概念。

## 文件结构

```
my-debug-module/
├── build.gradle                    # 模块构建文件（依赖Spring Session源码）
├── build-standalone.gradle         # 独立构建文件（不依赖源码）
├── README.md                       # 英文说明文档
├── 使用说明.md                     # 中文使用说明
└── src/
    ├── main/java/
    │   ├── SpringSessionDebugDemo.java    # 依赖Spring Session的演示程序
    │   └── SimpleSessionDemo.java          # 简化版演示程序（推荐使用）
    └── test/java/
        └── SimpleSessionTest.java         # 单元测试
```

## 快速开始

### 1. 运行简化版演示程序（推荐）

```bash
cd my-debug-module
javac -cp . src/main/java/SimpleSessionDemo.java
java -cp src/main/java SimpleSessionDemo
```

这个程序不依赖Spring Session源码，可以直接运行，展示了Session的核心概念。

### 2. 运行单元测试

```bash
cd my-debug-module
javac -cp . src/main/java/SimpleSessionDemo.java
javac -cp .:junit-4.11.jar src/test/java/SimpleSessionTest.java
java -cp .:junit-4.11.jar org.junit.runner.JUnitCore SimpleSessionTest
```

## 程序说明

### SimpleSessionDemo.java

这是一个完全独立的演示程序，模拟了Spring Session的核心概念：

**核心类：**
- `SimpleSession` - 模拟Session接口
- `SimpleSessionRepository` - 模拟SessionRepository接口

**演示功能：**
1. **Session创建** - 生成唯一ID，初始化时间戳
2. **属性操作** - 设置、获取、删除Session属性
3. **Session存储** - 模拟Session的保存和读取
4. **过期机制** - 演示Session的过期处理
5. **属性管理** - 展示Session属性的各种操作

## 常见问题

### Q: 为什么选择1.0.0版本？
A: 1.0.0是Spring Session的第一个稳定版本，代码相对简单，适合学习。新版本功能更复杂，但核心概念相同。

### Q: 如何理解Session过期机制？
A: Session过期是通过时间戳比较实现的。每次访问Session时更新lastAccessedTime，过期检查时比较当前时间与最后访问时间的差值。

### Q: 为什么需要SessionRepository？
A: SessionRepository是Session的存储抽象，允许使用不同的存储后端（内存、Redis、数据库等），提供了统一的Session管理接口。
